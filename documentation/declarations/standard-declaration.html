Standard Declaration

The StandardDeclaration offers a means of creating a JavaScript variable declaration from server-side code. 

	class StandardDeclaration : SuperScript.JavaScript.Declarables.DeclarationBase
	{
		bool AssignExisting { get; set; }
		override string ToString();
		object Value { get; set; }
	}
Name Type Description
AssignExisting bool

Indicates whether this declaration is new or is setting the value of an existing variable.

This value determines whether the emitted variable is preceded by the var keyword.

Optional. Default value: false

ToString string

Returns this instance as a formatted JavaScript declaration.

Value object

The value of the JavaScript declaration's source (i.e., the right-hand side of the declaration).

Inherited members
Comment string

Emitted inside JavaScript's multi-line comment symbols, a tab space after the end of the client-side declaration.

Optional.

EmitterKey string

Indicates which instance of IEmitter the content should be added to.

Optional. If not specified then the contents will be added to the default implementation of IEmitter.

Name string

Gets or sets the name assigned to this declarable.

Required.

Example

Variables may be declared in the config file.

Note: The examples on this page assume an instance of IEmitter has been configured with IEmitter.Key = "javascript". This is not configured by default. This page explains how to create and configure an IEmitter. 

	<configuration>
		<configSections>
			<section name="superScript" type="SuperScript.Configuration.SuperScriptConfig, SuperScript.Common" />
			<section name="superScript.JavaScript" type="SuperScript.JavaScript.Configuration.JavaScriptConfig, SuperScript.JavaScript" />
		</configSections>
	</configuration>
	

	…
	

	<superScript.JavaScript>
		<declarations>
			<variables>
				<variable name="exampleVar" value="hello world" comment="A commented variable" emitterKey="javascript" />
			</variables>
		</declarations>
	</superScript.JavaScript>

The above example will produce the following output: 

	var exampleVar = "hello world";		/* A commented variable */

When specified in the config file, the Type may also be declared. If not declared, as in the above example, the value is assumed to be a string.

The above example could therefore be changed accordingly. 

	<superScript.JavaScript>
		<declarations>
			<variables>
				<variable name="exampleVar" value="1" comment="A commented variable" emitterKey="javascript" type="System.Int32, mscorlib" />
			</variables>
		</declarations>
	</superScript.JavaScript>

The above example will produce the following output: 

	var exampleVar = 1;	/* A commented variable */

Variables may also be declared in code. SuperScript.JavaScript exposes a fluent syntax for declarables which allows for easier adding of declarations. In this case the Type is not specified. 

	var exampleVar = Declarations.AddVariable(opt => opt.Comment("A commented variable")
									.EmitterKey("javascript")
									.Name("exampleVar")
									.Value("hello world"));

The above example will produce the following output: 

	var exampleVar = "hello world";		/* A commented variable */

Alternatively, JavaScript variables may also be declared in a more conventional programmatic style. 

	var varDecl = new StandardDeclaration();
	varDecl.AssignExisting = false; // this is the default value
	varDecl.Comment = "A commented variable";
	varDecl.EmitterKey = "javascript";
	varDecl.Name = "exampleVar";
	varDecl.Value = 1;
	
	SuperScript.Declarations.AddDeclaration<StandardDeclaration>(varDecl);

The above example will produce the following output: 

	var exampleVar = 1;	/* A commented variable */
