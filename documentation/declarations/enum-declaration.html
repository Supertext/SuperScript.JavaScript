SuperScript offers a means of declaring assets in one part of a .NET web solution and have them emitted somewhere else.

Enum Declaration

The EnumDeclaration offers an easy way of transferring an enumeration from server-side code to client-side JavaScript.

It should be remembered that JavaScript does not strictly recognise an enum, however an object can be created which offers the same functionality as a conventional enum, and this is what is offered by the EnumDeclaration class. 

			class EnumDeclaration : SuperScript.JavaScript.Declarables.DeclarationBase
			{
				EnumAttribute EnumAttribute { get; set; }
				Type Type { get; set; }
				override string ToString();
			}
Name Type Description
EnumAttribute EnumAttribute

Allows alternate text to be specified for each enumeration.

Optional. Default value is EnumAttribute.None, which indicates that the enumeration text should be the same as the server-side enumeration text.

Type Type

The Type of the enumeration to be converted to JavaScript.

ToString string

Returns this enumerated type as a formatted JavaScript enumeration.

Inherited members
Comment string

This text may be emitted onto its own line or after a variable, depending upon the specific implementation of DeclarationBase.

Optional.

EmitterKey string

Indicates which instance of IEmitter the content should be added to.

Optional. If not specified then the contents will be added to the default implementation of IEmitter.

Name string

Gets or sets the name assigned to this declarable.

Optional. If not specified, SuperScript.JavaScript will use the name of the enumeration.

Example

SuperScript.JavaScript exposes a fluent syntax for declarables which allows for easier adding of declarations.

The following examples reference the following simple enumeration: 

			public enum DemoEnum
	        {
	            Value1,
	            Value2
	        }
		
			using SuperScript.JavaScript;
			

			Declarations.AddEnum(opt => opt.Emitter("javascript")
								.Name("DemoEnum")
								.Type(typeof (DemoEnum)));

A client-side name for the enumeration is not required to be specified. If not specified then SuperScript.JavaScript will use the server-side name of the enumeration for the name of its client-side counterpart.

Therefore, the following will produce the same output as above: 

			using SuperScript.JavaScript;
			

			Declarations.AddEnum(opt => opt.Emitter("javascript")
								.Type(typeof (DemoEnum)));

Alternatively, enumerations may also be declared in a more conventional programmatic style. 

		using SuperScript.JavaScript;
		

            	var enumDecl = new EnumDeclaration();
    		enumDecl.EmitterKey = "javascript";
    		enumDecl.Name = "DemoEnum";	// optional
    		enumDecl.Type = typeof(DemoEnum);

    		SuperScript.Declarations.AddDeclaration<EnumDeclaration>(enumDecl);

All of the above examples will output the following: 

			var DemoEnum = { "Value1": 0, "Value2": 1 };

Which can be used in the following way: 

			var enumVal = DemoEnum.Value1;	// assigns the value 0 to enumVal

Alternative Text

SuperScript.JavaScript also permits the textual enumerated type to be switched to an alternative value. This can be achieved by using the EnumAttribute attribute for each enumeration.

The DemoEnum used in these examples needs to be decorated: 

			public enum DemoEnum
	        {
	            [JsEnumText("value_one")
	            Value1,
	            
	            [JsEnumText("value_two")
	            Value2
	        }

To utilise the values in the JsEnumText attributes we can modify the above examples by specifying where the enumeration text should be obtained from. 

			using SuperScript.JavaScript;
			

			Declarations.AddEnum(opt => opt.Emitter("javascript")
								.EnumAttribute(EnumAttribute.EnumText)
								.Type(typeof (DemoEnum)));

Or using the conventional programmatic syntax: 

		using SuperScript.JavaScript;
		

            	var enumDecl = new EnumDeclaration();
    		enumDecl.EmitterKey = "javascript";
    		enumDecl.EnumAttribute = EnumAttribute.EnumText;
    		enumDecl.Name = "DemoEnum";	// optional
    		enumDecl.Type = typeof(DemoEnum);

    		SuperScript.Declarations.AddDeclaration<EnumDeclaration>(enumDecl);

The above examples will output the following: 

			var DemoEnum = { "value_one": 0, "value_two": 1 };

If no EnumAttribute is specified then the enumeration text in the client-side JavaScript will be the same as the server-side code.
