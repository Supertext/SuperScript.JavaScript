SuperScript offers a means of declaring assets in one part of a .NET web solution and have them emitted somewhere else.

This documentation is still being written. Please bear with us while we complete this documentation.

Array Declaration

By specifying a collection of objects, a JavaScript array may be declared. 

			class ArrayDeclaration : SuperScript.JavaScript.Declarables.DeclarationBase
			{
				bool AssignExisting { get; set; }
				object[] Elements { get; set; }
				override string ToString();
			}
Name Type Description
AssignExisting bool

Indicates whether this declaration is new or is setting the value of an existing variable.

This value determines whether the emitted variable is preceded by the var keyword.

Optional. Default value: false

Elements object[]

Specifies the elements of the array.

Required.

ToString string

Returns this instance as a formatted JavaScript declaration.

Type Type

Specify an explicit output type for the declared Elements.

If the array element type cannot be converted to the specified type, SuperScript.JavaScript will output the array element in its native, unconverted type.

Optional. Default value is System.String

Inherited members
Comment string

This text may be emitted onto its own line or after a variable, depending upon the specific implementation of DeclarationBase.

Optional.

EmitterKey string

Indicates which instance of IEmitter the content should be added to.

Optional. If not specified then the contents will be added to the default implementation of IEmitter.

Name string

Gets or sets the name assigned to this declarable.

Required.

It should be kept in mind that ArrayDeclaration inherits from DeclarationBase which in turn inherits from SuperScript.Declarables.DeclarationBase.

Example

Arrays may be declared in the config file.

Note: The examples on this page assume an instance of IEmitter has been configured with IEmitter.Key = "javascript". This is not configured by default. This page explains how to create and configure an IEmitter. 

			<configuration>
				<configSections>
					<section name="superScript" 
							 type="SuperScript.Configuration.SuperScriptConfig, SuperScript.Common, Version=0.0.0.1, Culture=neutral" />
					<section name="superScript.JavaScript" 
							 type="SuperScript.JavaScript.Configuration.JavaScriptConfig, SuperScript.JavaScript, Version=0.0.0.1, Culture=neutral" />
				</configSections>
			</configuration>
			

			…
			

			<superScript.JavaScript>
				<declarations>
					<arrays>
						<array name="demoArray" 
							   assignExisting="true"
							   comment="A comment"
							   elements="2014, 2015, 2016"
							   emitterKey="javascript"
							   type="System.Int32, mscorlib" />
					</arrays>
				</declarations>
			</superScript.JavaScript>

Arrays may also be declared in code. SuperScript.JavaScript exposes a fluent syntax for declarables which allows for easier adding of declarations. 

			using SuperScript.JavaScript;
			

			Declarations.AddArray(opt => opt.AssignExisting(false)
										    .Comment("A comment")
										    .Elements(new object[] { "2014", "2015", "2016" })
										    .EmitterKey("javascript")
										    .Name("demoArray")
                                            .Type(typeof(Int32));

Alternatively, arrays may also be declared in a more conventional programmatic style. 

			using SuperScript.JavaScript;
			

			var arr = new ArrayDeclaration();
            arr.AssignExisting = false;
            arr.Comment = "A comment";
            arr.Elements = new object[] { "2014", "2015", "2016" };
            arr.EmitterKey = "javascript";
            arr.Name = "demoArray";
            arr.Type = typeof (Int32);
			
            SuperScript.Declarations.AddDeclaration<ArrayDeclaration>(arr);

All of the above examples will output the following: 

			var demoArray = [2, 4, 6];	/* A comment */
