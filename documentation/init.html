Initialisation

In order for SuperScript.Javascript's internal properties to be initialised for each HTTP request, an additional step must be taken: for each HTTP request the default declarations should be added.

The default declarations are those which are declared in the config file or in the SuperScriptConfig class.

Here is an example showing how these default declarations could be added. 

  var defaultDecs = Configuration.Settings.Instance.Declarations;
  foreach (var declaration in defaultDecs)
  {
	  SuperScript.Declarations.AddDeclaration(declaration);
  }

There are two recommended locations for including this code, though these are only required if not installed via NuGet, or if the NuGet-added configuration is removed.

An HTTP handler with the above code is already included inside SuperScript.JavaScript. To enlist this HTTP handler the following code should be added to the web.config file: 

  <httpModules>
    <add name="HttpInitialiser" type="SuperScript.HttpInitialiser, SuperScript.JavaScript" />
  </httpModules>

An alternative is to use the Application_BeginRequest event in the project's global.asax file.
